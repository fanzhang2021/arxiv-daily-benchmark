1 Automatic Identification of Parallelizable Loops Using Transformer-Based Source Code Representations
📌 基于Transformer源代码表示的并行化循环自动识别
👤 Izavan dos S. Correia, Henrique C. T. Santos, Tiago A. E. Ferreira  |  📅 2026-04-01  |  🏷️ cs.SE, cs.AI
🔗 arXiv  ·  📄 PDF
💡 本文针对传统静态分析难以处理不规则代码以识别可并行循环的问题，提出了一种基于DistilBERT的Transformer方法，通过子词分词捕获代码上下文模式，实现了高精度的并行化潜力分类。
📋 展开中文摘要

在软件工程中，自动并行化仍然是一个具有挑战性的问题，特别是在识别代码区域中哪些循环可以在现代多核架构上安全地并行执行。传统的静态分析技术，如依赖分析和多面体模型，常常难以处理不规则或动态结构的代码。在这项工作中，我们提出了一种基于Transformer的方法来对源代码的并行化潜力进行分类，重点是区分独立（可并行化）循环与未定义的循环。我们采用DistilBERT，利用子词分词处理源代码序列，使模型能够捕获上下文句法和语义模式，而无需手工制作特征。该方法在一个结合了合成生成循环和人工标注真实世界代码的平衡数据集上进行了评估，使用了10折交叉验证和多种性能指标。结果显示出一致的高性能，平均准确率超过99%，且误报率低，证明了其鲁棒性和可靠性。与先前基于标记的方法相比，所提出的方法简化了预处理，同时提高了泛化能力并保持了计算效率。这些发现凸显了轻量级Transformer模型在循环级别实际识别并行化机会方面的潜力。

📋 展开中文摘要

聚合编程是一种基于场的协调范式，经过十多年的探索，已在传感器网络、机器人和物联网等多个领域成功应用，并拥有多种编程语言的实现，例如 Protelis、ScaFi (Scala) 和 FCPP (C++)。一个近期的研究方向将机器学习与聚合计算相结合，旨在支持大规模分布式学习，并为实现学习算法提供新的抽象。然而，现有的实现并未以数据科学实践者为目标，而他们主要使用Python——数据科学和机器学习的事实标准语言，拥有丰富且成熟的生态系统。Python在其他用例中也具有优势，例如教育和机器人学（例如通过ROS）。为了弥补这一空白，我们提出了Phyelds，一个用于聚合编程的Python库。Phyelds提供了功能齐全且轻量级的场演算计算模型实现，其特点是具有Pythonic的API和专为与Python机器学习生态系统无缝集成而设计的架构。我们描述了Phyelds的设计与实现，并通过从众所周知的聚合计算模式到联邦学习协调，以及与一个广泛使用的多智能体强化学习模拟器的集成，展示了其在多个领域的多功能性。

3 Think Anywhere in Code Generation
📌 代码生成中的随处思考
👤 Xue Jiang, Tianyu Zhang, Ge Li 等11人  |  📅 2026-04-01  |  🏷️ cs.SE, cs.LG
🔗 arXiv  ·  📄 PDF
💡 本文针对代码生成中前置思考的局限性，提出了一种让大语言模型能在生成代码的任何位置按需调用推理的新机制。
📋 展开中文摘要

推理大语言模型（LLMs）的最新进展主要依赖于前置思考，即推理发生在最终答案之前。然而，这种方法在代码生成中存在关键局限性，因为问题的全部复杂性往往在代码实现过程中才显现出来，前置思考通常是不够的。此外，它无法在难度差异显著的代码生成过程中自适应地分配推理精力。在本文中，我们提出了Think-Anywhere，一种新颖的推理机制，使LLMs能够在代码生成过程中的任何令牌位置按需调用思考。我们通过首先教导LLMs通过冷启动训练来模仿推理模式，然后利用基于结果的强化学习奖励来驱动模型自主探索何时何地调用推理，从而实现了Think-Anywhere。在四个主流代码生成基准（即LeetCode、LiveCodeBench、HumanEval和MBPP）上进行的大量实验表明，Think-Anywhere在现有推理方法和最近的训练后方法上都达到了最先进的性能，同时在不同LLMs上展示了一致的泛化能力。我们的分析进一步揭示，Think-Anywhere使模型能够自适应地在高熵位置调用推理，提供了更强的可解释性。

5 SkillReducer: Optimizing LLM Agent Skills for Token Efficiency

💡 本文针对LLM代码智能体技能描述冗长、内容低效导致令牌成本高和注意力分散的问题，提出了一个两阶段优化框架SkillReducer来压缩和重构技能内容。
📋 展开中文摘要

基于LLM的编码智能体依赖于*技能*——这些预先打包的指令集扩展了智能体的能力，然而注入到上下文窗口中的每一个技能内容令牌都会产生货币成本并导致注意力稀释。为了理解这个问题的严重性，我们对55,315个公开可用的技能进行了大规模实证研究，发现了系统性的低效问题：26.4%的技能完全缺乏路由描述，超过60%的主体内容是不可操作的，并且每次调用时参考文件可能注入数万个令牌。基于这些发现，我们提出了SkillReducer，一个两阶段优化框架。阶段1通过压缩冗长的描述和通过对抗性增量调试生成缺失的描述来优化路由层。阶段2通过分类学驱动的分类和渐进式披露来重构技能主体，将可操作的核心规则与按需加载的补充内容分离开来，并通过忠实度检查和自校正反馈循环进行验证。在600个技能和SkillsBench基准测试上进行评估，SkillReducer实现了48%的描述压缩和39%的主体压缩，同时将功能质量提高了2.8%，揭示了一种*少即是多*的效应，即移除非必要内容减少了上下文窗口中的注意力分散。这些益处跨四个系列的五个模型得以转移，平均保留率为0.965，并且可以推广到一个独立的智能体框架。

📋 展开中文摘要

部署在受控分布式环境中微型计算单元上的模块化软件通常需要两种消息传递路径：低开销的进程内协调和选择性的跨节点分发。在实践中，事件标识、序列化和传输桥接经常被实现为临时性的粘合代码，这使得进程间通信（IPC）、结构化路由和关闭行为变得复杂。本文提出了CNS，一种以异步“发射后不管”消息传递为中心的轻量级本地优先混合事件结构。CNS结合了类型化事件键、按族序列化和验证、用于进程内协调的本地发布/订阅上下文，以及一个由NATS支持的用于节点间分发的分布式上下文。一个桥接运行时在两个上下文之间移动事件，同时保留一个共同的路由词汇表。主要的操作模式是“发射后不管”的发布和订阅；在同一主题空间上，双向请求-回复仍可作为次要扩展使用。本文报告了一个Python原型和单机测量结果。纯本地传递平均耗时约30微秒。纯分布式传递平均耗时1.26-1.37毫秒，混合桥接平均耗时1.64-1.89毫秒。相对于序列化选择，验证引入了适度的开销。最终构建的产物适用于结构化IPC，以及模块化服务内部和受控节点有限集合之间的实际消息移动。
